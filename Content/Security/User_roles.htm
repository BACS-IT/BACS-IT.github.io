User roles

User roles determine the areas and features of Act! that a user can access or utilize.

Act! includes five roles: This is the most powerful role assigned, giving the user access to all areas of the application and to all data in the supplemental files. The administrator also has access to all management tools, such as database maintenance and synchronization.Administrator, This role is the second most powerful user role, and is designed for users who require system wide access with the exception of management tools. Manager, This role is the most common user role and is designed for users who require access to their own records and data, as well as other users records. This is the default user role. Standard, This role allows users to access areas of the application that he or she has been granted permission to. Restricted, and This role is the most limited and only allows users to view calendars and to add activities. A Browser user cannot modify or add any other data in the database.Browse. A user can only be assigned one role. Each role is given permission to access one or more areas of the database.

The areas, such as creating or deleting a contact, are combined in feature sets. Feature sets are pre-defined and specify which feature can be accessed by each role. For example, the standard user role is given permission to create, edit, and delete a contact record as defined in the "Contacts" feature set. The restricted user role, however, may only be granted permission to create and edit contact records.  For detailed information about each role, see the Act! online Help.
